Introduction to HTML

HTML stands for HyperText Markup Language. It is the fundamental language used to create and structure content on the web. HTML uses tags (also called elements) to define parts of a webpage such as headings, paragraphs, links, lists, and more.

What is Markup?

"Markup" means that you are marking up the content to give it structure and meaning. For example, wrapping a sentence with <p> tells the browser: "This is a paragraph."

What is a Web Page Made of?

A basic HTML document includes:

  • A doctype declaration (<!DOCTYPE html>) which tells the browser to use the latest HTML standard.
  • A root <html> element that contains everything.
  • A <head> section with metadata, title, and links to styles or scripts.
  • A <body> section with the visible content of the page.

Example:

<!DOCTYPE html>
    <html>
        <head>
            <title>My First Webpage</title>
        </head>
        <body>
            <h1>Welcome to My Website</h1>
            <p>This is a simple HTML document.</p>
        </body>
  </html>

Common HTML Tags

HTML tags are the building blocks of your webpage. Most tags come in pairs: an opening tag like <p> and a closing tag like </p>. Tags that don't require closing tags are called void elements.

Some essential tags:
Tag Description
<h1></h1> to <h6></h6> Headings (from most important to least)
<p></p> Paragraph of text
<a href=""></a> Anchor tag for links
<img src=""> Displays an image
<ul></ul> and <ol></ol> Unordered and Ordered lists
<li></li> List item
<br> Line break
<hr></hr> Horizontal line (divider)
<strong></strong> and <em></em> Bold and italic for emphasis

Example:

<h2>Learning HTML</h2>
<p>This is a paragraph with a <strong>bold</strong> word.</p>
<a href="https://developer.mozilla.org">Visit MDN</a>
<img src="html-logo.png" alt="HTML5 Logo">

Forms and Input Elements

HTML forms allow users to input data, such as filling out a contact form or logging into a site.

Basic Form Structure:

<form action="/submit" method="POST">
    <!-- input elements go here -->
</form>
  • action: URL where the form data will be sent
  • method: usually "GET" or "POST"

Common Form Elements:
Element Purpose
<input type="text"> Single-line text input
<input type="email"> Email input with built-in validation
<input type="password"> Hidden password field
<textarea> Multi-line input
<select> + <option> Dropdown menu
<input type="submit"> Button to submit the form

Example:

<form>
    <label for="username">Username:</label><br>
    <input type="text" id="username" name="username"><br><br>

    <label for="email">Email:</label><br>
    <input type="email" id="email" name="email"><br><br>

    <label for="message">Message:</label><br>
    <textarea id="message" name="message" rows="4" cols="30"></textarea><br><br>
    
  <input type="submit" value="Send">
</form>

Semantic HTML Elements

Semantic HTML refers to elements that clearly describe their meaning in a human- and machine-readable way. Instead of using generic <div>s everywhere, semantic tags improve the structure and accessibility of your page.

Common Semantic Tags:
Tag Purpose
<header> Intro or navigation for a section or page
<nav> Navigation menus
<main> Main content of the page
<section> Thematic grouping of content
<article> Self-contained content (e.g. blog post)
<aside> Side content (e.g. ads, tips)
<footer> Footer content

Example:

<main>
    <article>
        <header>
            <h2>Why Semantic HTML Matters</h2>
        </header>
        <p>It improves accessibility, SEO, and code clarity.</p>
        <footer>Written by John</footer>
    </article>
</main>

Using semantic elements makes your code easier to maintain and more meaningful to users and search engines.

HTML Best Practices

Writing good HTML means not just making a page work, but also making it clean, readable, and future-proof.

General Guidelines:

  • Use semantic elements instead of too many <div>s.
  • Indent your code consistently.
  • Use lowercase for tag names and attributes.
  • Always close your tags, even if they're optional.
  • Add alt attributes to images for accessibility.
  • Avoid inline styling; use CSS instead.
  • Test your HTML in different browsers.

Examples:

Bad:
<DIV><P style="color:blue">text</P></DIV>
Good:
<div>
    <p class="blue-text">text</p>
</div>
Accessibility:
<img src="form.jpg" alt="Filling out a form on a laptop">

This helps users with screen readers know what the image is about.
